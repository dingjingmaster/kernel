# 前言

可执行和链接格式规范1.2版本由TIS（Tool Interface Standards）制定，TIS一个由微型计算机行业成员组成的协会，致力于32位英特尔体系结构操作环境中开发工具可见的软件接口的标准化。这些接口包括对象模块格式、可执行文件格式、调试记录信息和格式。

该委员会的目标是帮助简化整个微型计算机工业的软件开发过程，目前集中于32位操作环境。为此，委员会制定了一些规范——一些文件格式适用于主要的工业操作系统，另一些描述了32位Windows操作系统的格式。最初作为TIS可移植格式规范1.1版共同发布，这些规范现在被分开并单独发布。

TIS委员会成员包括来自Absoft、Autodesk、Borland国际公司、IBM公司、Intel公司、Lahey公司、Lotus公司、MetaWare公司、Microtec Research公司、微软公司、Novell公司、The Santa Cruz Operation公司和WATCOM国际公司的代表。PharLap Software Incorporated和赛门铁克公司也参与了规范定义工作。

与TIS规范集合中的其他规范一样，该规范基于现有的、经过验证的格式，与TIS委员会的目标保持一致，即采用并在必要时扩展现有标准，而不是发明新标准。

本章适用于在各种平台上创建对象或可执行文件的32位环境操作系统开发者。

## 关于ELF(Executable and Linking Format)

可执行和链接格式最初是由UNIX系统实验室（USL）作为应用程序二进制接口（ABI）的一部分开发和发布的。工具接口标准委员会（TIS）已经选择了不断发展的ELF标准作为可移植的目标文件格式，该格式适用于各种操作系统的32位Intel体系结构环境。

ELF标准旨在通过向开发人员提供一组跨多个操作环境扩展的二进制接口定义来简化软件开发。这将减少不同接口实现的数量，从而减少重新编码和重新编译代码的需要。
