# /proc/<pid>/maps

用于显示指定进程的虚拟内存映射信息。

## 文件内容格式

`/proc/<pid>/maps`每一行描述一个虚拟内存区域(Virtual Memory Area, VMA)，格式如下：
```
address     perms       offset      dev     inode   pathname
```
- address：地址范围(格式：start-end，不包含结束地址)
- perms：权限（r: 可读、w: 可写、x: 可执行、p: 私有,写时复制、s: 共享,修改后会反映到映射文件中、-: 表示没有权限）
- offset：偏移量，对于文件映射的内存区域，表示该区域映射的文件内容的起始偏移量(相对于文件开头、单位是字节)。如果该区域不是文件映射(例如：堆或栈)，偏移量为0
- dev：设备号，表示映射文件所在设备的设备号
- inode：索引节点，表示映射文件的索引节点号。如果是0,表示该区域不是文件映射(例如：匿名内存、堆、栈等)
- pathname：文件路径或特殊标记。匿名映射(空字符串)、`[heap]`进程堆内存、`[stack]`进程主线程栈、`[vdso]`虚拟动态共享对象(vDSO)，用于加速系统调用、`[vvar]`与vDSO相关的变量数据、`[vsyscall]`旧式系统调用接口、`[annon:<name>]`命名匿名映射
